# Система комиссий маркетплейсов

## Обзор

Система комиссий MP Control хранит, обновляет и использует точные ставки комиссий Wildberries и Ozon для расчёта плановой и фактической прибыли по заказам.

## Архитектура

### Таблицы БД

| Таблица | Назначение |
|---------|-----------|
| `marketplace_commission_versions` | Версии тарифных наборов (WB/Ozon) с датами действия |
| `marketplace_commission_rates` | Нормализованные ставки комиссий по категориям/моделям продаж/ценовым диапазонам |
| `marketplace_tariff_source_checks` | Журнал проверок официальной страницы Ozon на обновления |
| `marketplace_commission_import_logs` | Журнал импорта XLSX-файлов комиссий Ozon |

### Поля order_items для отслеживания комиссий

| Поле | Назначение |
|------|-----------|
| `commission_estimated` | Рассчитанная сумма комиссии |
| `commission_source` | Источник комиссии (wb_tariff_db, ozon_tariff_db, financial_actual и т.д.) |
| `commission_percent_planned` | Процент комиссии из справочника |
| `commission_amount_planned` | Сумма комиссии, рассчитанная от базы |
| `commission_rate_version_id` | Ссылка на версию тарифа |
| `commission_rate_id` | Ссылка на конкретную ставку |
| `commission_match_status` | exact / not_found |
| `commission_calculation_confidence` | exact / estimated / not_available |
| `ozon_commission_base_price` | База расчёта комиссии Ozon (цена продавца) |

### Сервисы

| Сервис | Файл | Назначение |
|--------|------|-----------|
| `WbCommissionSyncService` | `wb_commission_sync_service.py` | Синхронизация комиссий WB через официальный API |
| `OzonCommissionSourceMonitorService` | `ozon_commission_source_monitor_service.py` | Мониторинг страницы Ozon на новые таблицы комиссий |
| `OzonCommissionXlsxImporter` | `ozon_commission_xlsx_importer.py` | Валидация и импорт XLSX-файлов комиссий Ozon |
| `CommissionResolverService` | `commission_resolver_service.py` | Единый резолвер ставок комиссий для расчётов |

## Wildberries

### Автоматическая синхронизация

- **Расписание**: ежедневно в 03:10 МСК
- **Источник**: `GET /api/v1/tariffs/commission` (common-api.wildberries.ru)
- **Логика**:
  1. Запрос актуальных тарифов через API
  2. Сравнение SHA256 хеша с последней активной версией
  3. Если данные изменились — создаётся новая активная версия, предыдущая закрывается
  4. Если данных нет изменений — только логирование
  5. Администратор получает уведомление при изменении или ошибке

### Ручной запуск

- **Telegram**: Админ-меню → Комиссии маркетплейсов → Обновить комиссии WB
- **WEB**: `/web/admin/commissions` → кнопка «Обновить комиссии WB»

## Ozon

### Методология расчёта вознаграждения Ozon

Согласно официальным правилам Ozon, вознаграждение за продажу рассчитывается:

1. **База расчёта** (`ozon_commission_base_price`):
   - От установленной продавцом цены товара (`product.price` в Ozon API);
   - Цена включает применимый НДС;
   - Учитываются все скидки и акции, заданные продавцом;
   - НЕ учитываются дополнительные скидки за Баллы Ozon.

2. **Определение ставки**:
   - По категории товара;
   - По типу товара;
   - По схеме продаж: FBO / FBO Fresh / FBS / RFBS;
   - По ценовому диапазону, рассчитанному от `ozon_commission_base_price`.

3. **Формула расчёта**:
   ```
   commission_amount = ozon_commission_base_price × commission_percent / 100
   ```

4. **Время применения ставки**:
   - По ставке, действовавшей на момент оформления заказа;
   - Фактически списывается при переходе товара в статус "Доставлен".

5. **Возвраты**:
   - При возврате товара Ozon возвращает ранее списанное вознаграждение;
   - Фактическая комиссия корректируется через финансовые данные.

### Определение базы расчёта Ozon

| Поле Ozon API | Использование |
|---------------|--------------|
| `product.price` | **Основная база** — цена продавца, установленная для товара |
| `financial_data.commission_amount` | Фактическая комиссия (приоритетнее справочника) |
| `financial_data.payout` | Выручка продавца после всех расходов |

**Стратегия**:
- Если `product.price` доступен → `ozon_commission_base_price = product.price`, confidence = **exact**
- Если только `financial_data` доступен без `product.price` → используем доступную цену, confidence = **estimated**
- Если нет достоверных данных → confidence = **not_available**, комиссия не рассчитывается

### Мониторинг обновлений

- **Расписание**: ежедневно в 03:30 МСК
- **Источник**: https://seller-edu.ozon.ru/libra/commissions-tariffs/commissions-tariffs-ozon/komissii-tovary-uslugi
- **Логика**:
  1. Загрузка страницы и поиск актуального блока с периодом комиссий
  2. Извлечение ссылки на XLSX-файл «Скачать таблицу категорий»
  3. Сравнение с последней успешной проверкой
  4. При обнаружении изменений — уведомление администратору
  5. **Автоматический импорт НЕ выполняется** — требуется ручная загрузка

### Загрузка XLSX через WEB

1. Перейти на `/web/admin/commissions`
2. Заполнить форму:
   - Файл XLSX
   - Дата начала действия тарифов (ДД.ММ.ГГГГ)
   - Название версии (опционально)
3. Нажать «Загрузить и импортировать»
4. Система валидирует файл, импортирует ставки и активирует новую версию

### Загрузка XLSX через Telegram-бот

1. Админ-меню → Комиссии маркетплейсов → Загрузить таблицу комиссий Ozon
2. Отправить XLSX-файл боту
3. Указать дату начала действия в формате ДД.ММ.ГГГГ
4. Бот валидирует, импортирует и отправляет подробный результат

### Валидация XLSX

Импортер проверяет:
- Расширение файла (.xlsx)
- Наличие листа «Прайс РФ (БЗ)»
- Наличие колонок «Категория» и «Тип товара»
- Наличие групп продаж (FBO, FBS, RFBS)
- Корректность значений комиссий (0..1 или 0..100, нормализуются в проценты)
- Дубликат по SHA256 (повторный импорт того же файла блокируется)

## Использование в расчётах

### Приоритет источников комиссии

1. **Фактическая комиссия** из финансовых отчётов маркетплейса (если доступна)
2. **Ставка из продукта** (поля `commission_fbw`, `commission_fbs` и т.д., сохранённые при синхронизации продукта)
3. **Справочник тарифов** (CommissionResolverService по БД)
4. **Не определена** — если ничего не найдено

### CommissionResolverService

```python
result = await resolver.get_commission_rate(
    marketplace="OZON",
    order_date=date(2026, 5, 20),
    sales_model="fbs",
    category_name="Одежда",
    product_type_name="Футболка",
    ozon_commission_base_price=Decimal("1500"),  # цена продавца
)
```

Возвращает `CommissionResolutionResult`:
- `commission_percent` — ставка в процентах или None
- `commission_base_price` — база расчёта (для Ozon)
- `commission_amount` — рассчитанная сумма комиссии (для Ozon)
- `match_status` — "exact", "not_found"
- `calculation_confidence` — "exact", "estimated", "not_available"
- `source` — источник ставки
- `diagnostics` — пояснение

### Уровни уверенности расчёта (confidence)

| Уровень | Когда применяется | Отображение |
|---------|------------------|-------------|
| `exact` | База расчёта определена точно (product.price из Ozon API) | "💼 Комиссия Ozon: 82 ₽ (27%, тариф Ozon)" |
| `estimated` | Данные API не позволяют идеально воспроизвести формулу, но есть близкая оценка | "💼 Комиссия Ozon: около 82 ₽ (27%, оценка)" |
| `not_available` | Корректно посчитать нельзя | "💼 Комиссия Ozon: не определена — недостаточно данных" |

### Отображение в уведомлениях

- Если ставка найдена (exact): `💼 Комиссия маркетплейса: 47 ₽ (16%, тариф WB)`
- Если ставка найдена (Ozon, exact): `💼 Комиссия Ozon: 82 ₽ (27%, тариф Ozon)`
- Если ставка оценочная: `💼 Комиссия Ozon: около 82 ₽ (27%, оценка)`
- Если ставка не найдена: `💼 Комиссия маркетплейса: не определена — тариф не найден`

## Плановая vs фактическая комиссия

- **Плановая** — рассчитывается на момент создания заказа из справочника тарифов. Используется для предварительного расчёта прибыли в уведомлениях.
- **Фактическая** — берётся из финансовых отчётов маркетплейса после их получения. Приоритетнее плановой.

## Возвраты и фактическая аналитика Ozon

Согласно официальным правилам Ozon, при возврате товара вознаграждение за продажу возвращается продавцу.

**Текущая реализация**:
- Возвраты Ozon синхронизируются через `/v1/returns/list` и сохраняются в `returns_events`
- Фактические финансовые данные Ozon (`financial_data.commission_amount`) содержат корректную комиссию с учётом возвратов
- При наличии финансовых данных фактическая комиссия приоритетнее плановой

**Ограничение**:
- Для полной точности возвратов требуется отдельная доработка финансовых событий Ozon, чтобы явно связывать возврат комиссии с конкретным заказом
- В текущей реализации возврат комиссии учитывается через общие финансовые данные, но не через явную операцию «возврат комиссии»

## Соответствие официальным правилам Ozon

### Как определена база расчёта

`ozon_commission_base_price` = `product.price` из Ozon Posting API — это цена, установленная продавцом, которая включает НДС и скидки продавца, но не включает скидки за Баллы Ozon.

### Какие поля Ozon API используются

| Поле | Источник | Назначение |
|------|---------|-----------|
| `product.price` | `/v3/posting/fbs/list`, `/v2/posting/fbo/list` | База расчёта комиссии |
| `financial_data.commission_amount` | financial_data в posting | Фактическая комиссия |
| `financial_data.payout` | financial_data в posting | Выручка продавца |
| `financial_data.services` | financial_data в posting | Логистика и другие услуги |

### Когда расчёт exact / estimated / not_available

| Confidence | Условие |
|-----------|---------|
| `exact` | `product.price` доступен + ставка найдена в справочнике |
| `estimated` | `product.price` недоступен, используется fallback цена + ставка найдена |
| `not_available` | Ставка не найдена в справочнике ИЛИ нет данных о цене |

### Насколько текущий расчёт соответствует официальным правилам

- ✅ База расчёта = цена продавца (product.price)
- ✅ Ставка определяется по категории, типу, схеме продаж, ценовому диапазону
- ✅ Версия тарифов выбирается по дате оформления заказа
- ✅ Формула: base_price × percent / 100
- ⚠️ Скидки Ozon Points не вычитаются из базы (требует дополнительных данных из analytics_data)
- ⚠️ НДС включён в price, но не выделяется отдельно (соответствует правилам Ozon)

### Ограничения и следующий этап

1. **Ozon Tariff Engine** — расширить систему до расчёта всей экономики заказа:
   - Логистика (зависит от габаритов, региона, схемы)
   - Последняя миля
   - Обратная логистика
   - Обработка возвратов/невыкупов/отмен
   - Эквайринг
   - Дополнительные услуги

2. **Точное определение скидок Ozon Points** — требуется парсинг `analytics_data` posting для выделения скидок, не влияющих на комиссию

3. **Явный возврат комиссии при возвратах** — связывать финансовые операции возврата комиссии с конкретными заказами для точной P&L аналитики

4. **Исторические тарифы** — при загрузке XLSX Ozon автоматически определять период действия из названия файла, а не требовать ручного ввода даты
